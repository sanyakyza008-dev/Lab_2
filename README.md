## 1. Описание задачи
Необходимо перемножить 2 квадратные матрицы размера 4096x4096 с элементами типа `float`. 
Исходные матрицы генерируются случайным образом в диапазоне от -1.0 до 1.0 и сохраняются в бинарные файлы с помощью вспомогательной программы `MatrixGenerator`.

Для оценки эффективности алгоритмов используются следующие формулы:
* **Сложность алгоритма (количество операций):** $c = 2n^3$, где $n$ - размерность матрицы (4096).
* **Производительность (MFlops):** $p = \frac{c}{t} \cdot 10^{-6}$, где $t$ - время работы алгоритма в секундах.

В работе реализовано и проанализировано 3 варианта перемножения матриц.

## 2. Инструкция по сборке проекта

Проект настроен с использованием системы сборки CMake. В папке `lib` находится сжатая библиотека OpenBLAS для Windows-систем, которая **распакуется и скопируется в нужную директорию автоматически** на этапе сборки.

**Сборка через Visual Studio IDE**
1. Откройте Visual Studio и выберите на стартовом экране **«Открыть локальную папку»** (Open a local folder), указав папку проекта.
2. Дождитесь, пока Visual Studio автоматически прочитает `CMakeLists.txt` и завершит конфигурацию кэша.
3. На верхней панели инструментов выберите конфигурацию **`x64-Release`** (это необходимо для включения аппаратных оптимизаций компилятора MSVC).
4. В верхнем меню выберите **«Сборка» -> «Собрать все»** (Build -> Build All).
5. На верхней панели в выпадающем списке запуска (рядом с зеленым треугольником) выберите **`MatrixGenerator.exe`** и запустите его для генерации файлов `matrix_A.bin` и `matrix_B.bin`.
6. Переключитесь в этом же списке на **`calculator.exe`** и запустите вычисления.

**Сборка через консоль**
1. Сгенерируйте файлы сборки:
   `cmake -B build`
2. Выполните компиляцию проекта с максимальной оптимизацией:
   `cmake --build build --config Release`
3. Сначала запустите генератор матриц (путь зависит от используемого компилятора):
   - Для MSVC: `build\Release\MatrixGenerator.exe`
   - Для MinGW/GCC: `build\MatrixGenerator.exe`
4. Затем запустите основной алгоритм вычислений:
   - Для MSVC (Windows): `build\Release\calculator.exe`
   - Для MinGW/GCC: `build\calculator.exe`

## 3. Анализ алгоритмов перемножения

### Вариант 1: Наивный алгоритм
Стандартный алгоритм перемножения матриц с использованием трех вложенных циклов.
Алгоритм имеет вычислительную сложность $O(N^3)$ и абсолютно не учитывает архитектуру кэш-памяти процессора.
* **Результат:** Пропущен при тестировании больших матриц. Из-за неоптимального доступа к памяти выполняется очень долго.

### Вариант 2: Использование библиотеки BLAS
Для эталонного тестирования используется функция `cblas_sgemm` из библиотеки OpenBLAS.
* **Время выполнения:** ~3.4052 с. 
* **Производительность:** ~40364 MFlops. 

### Вариант 3: Оптимизированный алгоритм

1. **Изменение порядка циклов (i-k-j):** Обеспечивает последовательный доступ к элементам матриц, что значительно снижает количество кэш-промахов.
2. **Блочное умножение (Tiling):** Матрицы разбиты на блоки размером 64x64 элемента (16 Кбайт). Это позволяет данным полностью помещаться в сверхбыстрый кэш L1 процессора (обычно 32-48 Кбайт на ядро).
3. **Параллелизм:** Использована технология OpenMP (`#pragma omp parallel for`) для распределения вычислений независимых блоков между ядрами процессора.
4. **упаковка данных против Cache/TLB Thrashing**: Упаковка подматрицы в непрерывный буфер в памяти минимизирует промахи TLB и устраняет конфликты ассоциативности кэша.

* **Время выполнения:** ~3.3008 с. 
* **Производительность:** ~41753 MFlops.
